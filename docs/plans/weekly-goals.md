# Weekly Goals

A quick overview of what to accomplish each week.

---

## Week 1: Dream It (Days 1-4)

**Goal:** Decide what game we're making and write it all down.

**Key Deliverables:**
- Project named
- 4 PRDs completed (concept, characters, story, gameplay)
- Team roles assigned
- Inquiry question locked

**Success looks like:** Everyone agrees on what the game is, and it's all written down in PRDs.

---

## Week 2: Learn It (Days 5-9)

**Goal:** Finish all design docs and write your first code.

**Key Deliverables:**
- All 8 PRDs reviewed and approved
- Python basics learned (variables, loops, if statements)
- First game window on screen
- Controller input working
- Player rectangle moves with controller

**Success looks like:** A window with a moving rectangle that responds to the Nintendo controller.

---

## Week 3: Build It (Days 10-14)

**Goal:** Build the actual game.

**Key Deliverables:**
- Core mechanic working and fun
- Player sprite (real art, not rectangles)
- At least 1 enemy type
- Scoring or collectibles
- Start screen, game over, restart
- Sound effects
- At least 1 level or endless mode

**Success looks like:** A playable game from start to finish, even if it's rough.

---

## Week 4: Ship It (Days 15-17)

**Goal:** Polish the game and prepare for the exhibition.

**Key Deliverables:**
- All bugs fixed
- Difficulty balanced
- Code frozen (v1.0)
- Exhibition demo station ready
- Learning stories written

**Success looks like:** Someone walks up at the exhibition, picks up the controller, and plays your game. And they have fun.

---

## Tips for Staying on Track

1. **Check the master plan daily** — what's the next unchecked item?
2. **Don't skip PRDs** — Week 1 design work saves Week 3 pain
3. **Ask for help early** — stuck for 10 minutes? Ask Claude or Mark
4. **Simple is better** — a finished simple game beats an unfinished complex one
5. **Log your learning** — the exhibition cares about your JOURNEY, not just the game
